## Considered and rejected findings

The following findings were identified during the audit but NOT written as plans
for the reasons given:

1. **`src/app/api/admin/verify/route.ts:45` cleartext password comparison**
   (`password === adminPassword`). The comparison is inside a POST handler that
   already has rate limiting (5 req/5min). The admin session is HMAC-signed and
   compared with `timingSafeEqual`. Upgrading the login comparison to
   `timingSafeEqual` is good hygiene but the endpoint is already scoped (rate
   limited, not a route anyone brute-forces) and the session itself has the real
   protection. ~Low risk, low leverage. Not worth the plan.~

2. **`hardSellConfig` fallback at `ResultPage.tsx:53`** silently maps unknown
   archetype slugs to `curious-professional`. This is a design defense (what if
   someone bookmarks a result page for a stale slug?). The scoring function
   always produces one of 4 known slugs, so this only matters if a slug is
   removed from `archetypes.ts`. ~Defer to when archetypes change.~

3. **`avgScore` unused variable at `ResultPage.tsx:54`**. Dead code, trivial
   cleanup. A single delete. ~Too small for a plan; include in a general
   cleanup pass.~

4. **`img-src 'https:'` CSP wildcard** at `next.config.ts:74`. The CSP
   allows images from any HTTPS source. In practice, the Next.js image
   component hardcodes allowed hosts in `images.remotePatterns`. The CSP
   wildcard is a bypass of that restriction but narrowing it would require
   auditing every image source on the site. ~Defer to a broader CSP hardening
   pass.~

5. **Admin routes lack middleware guard**. The `/admin/verify` endpoint exists
   but there's no middleware protecting all `/admin/*` routes. No admin UI
   pages exist beyond the verify endpoint, so there's nothing to protect.
   ~Defer until admin routes are actually built.~

6. **Next.js 16.1.6 has multiple CVEs** (12 advisories including high-severity
   cache poisoning, CSRF bypass, SSRF). Upgrading Next.js and the Firebase
   dependencies would fix these. This is a high-value upgrade but requires a
   separate branch, smoke test of all pages, and coordination with the Firebase
   App Hosting deploy pipeline. ~Out of scope for this plan set; recommended as
   a standalone migration plan.~

7. **No pre-commit hooks (husky, lint-staged)**. Tooling gap but the CI
   workflow runs `npm run lint` on push/PR. ~Nice-to-have, not a blocker.~

8. **`.env.local` contains real values (armchairfuturist@gmail.com)**. The
   file is gitignored. The email address is a public business contact.
   ~Not a secret.~

9. **In-memory rate limiter** at `email-utils.ts` is ephemeral -- doesn't
   survive serverless cold starts, not shared across Cloud Run instances. This
   is a known limitation in the ADR's email strategy. True rate limiting at
   Cloud Run scale requires Redis or Firestore-based limiting. ~Low leverage
   for this project's traffic scale.~
